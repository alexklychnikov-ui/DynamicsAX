НИКОГДА не удалять существующий код. Если он не нужен, то комментируем его по правилам в commentrules
ВАЖНО при сохранении файлов в формате XPO как можно меньше изменять исходный файл

commentrules
все комментарии выполняются по следующим правилам:
формат комментария // a.klychn 04.02.2025 MIK_PUR_000002017_01 Где 
 - a.klychn это имя пользователя
 - 04.02.2025 текущая дата
 - MIK_PUR_000002017_01 наименование проекта
комментарий для одной исправленой/вставленной строки делается через пробел от последнего символа строки
для вставленного/измененного блока (несколько сирок подряд) кода делается по следующему шаблону
// + a.klychn 04.02.2025 MIK_PUR_000002017_01
изменненый код 
изменненый код 
// - a.klychn 04.02.2025 MIK_PUR_000002017_01
с сохранением количества ведущих знаков таблуляции у измененного кода
если создается новый класс то комментарий ставится первой строкой в методе declareclass 
если создается новый метод в классе/таблице/форме и т.п. то комментарий ставится первой строкой в методе
