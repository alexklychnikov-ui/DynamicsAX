# XPO: распарсить в parserXPO

В корне репозитория DynamicsAX выполни парсинг XPO в дерево `parserXPO/<Объект>/<метод>.xpp`.

- Команда: `python xpo_parser.py <путь_к.xpo> [parserXPO] [--force] [--no-input]`
- По умолчанию выход: `parserXPO`. Флаг `--force` перезаписывает существующие `.xpp`.
- В папке `XPO` при интерактивном запуске без аргументов не предлагай обрабатывать файлы `*_WR.xpo` (они уже результат `xpo_writer`).
- Для агентов/CI: `--no-input` или `CI=1` / `XPO_NO_INPUT=1` — без диалога проекта; если в `XPO` несколько `.xpo`, обрабатываются все.
- Точечное извлечение из большого экспорта по SQLite: `parse_object_from_index(...)` в `xpo_parser.py` и индекс `indexXPO_cus/xpo_index.db`.

После выполнения кратко перечисли созданные/обновлённые каталоги объектов.
