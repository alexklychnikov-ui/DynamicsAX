# XPO: собрать _WR.xpo из parserXPO

Запиши изменённые `.xpp` из `parserXPO` обратно в XPO (новый файл рядом с исходником).

- Команда: `python xpo_writer.py <путь_к_исходному.xpo> [parserXPO] [--force] [--no-input]`
- Результат: `XPO/<stem>_WR.xpo` (или тот же каталог, где лежит исходный `.xpo`).
- Без `--force` подставляются только методы, у которых `.xpp` новее mtime исходного XPO (или отличается текст).
- `--force` — считать все `.xpp` кандидатами на подстановку.
- `--no-input` / `CI=1` / `XPO_NO_INPUT=1` — не спрашивать про добавление отсутствующих в XPO элементов по шаблону; несколько `.xpo` в `XPO` без аргументов — обработать все.
- Не правь `_WR.xpo` как первичный источник; источник правок — `parserXPO`.

Сообщи путь к `_WR.xpo` и сколько методов обновлено (из вывода скрипта).
