# XPO: roundtrip после правок XPP

Пользователь правил X++ в `parserXPO`. Сделай контрольный проход:

1. Запусти `python xpo_writer.py <исходный.xpo> [parserXPO] [--force]` при необходимости с `--no-input` в неинтерактивной среде.
2. Убедись, что появился/обновился файл `<stem>_WR.xpo` рядом с исходным `.xpo`.
3. Краткий чеклист для импорта в AOT: открыть `_WR.xpo` в AX, проверить конфликты слоя, прогнать компиляцию затронутых объектов.
4. Если нужен обратный разбор уже собранного файла — парсить **исходный** `.xpo` или новый экспорт, а не править `_WR.xpo` руками как единственный источник.

Не запускай полный парсинг гигантского CUS-экспорта без индекса/MCP — см. правила проекта.
